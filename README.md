# Intern Edu — Flutter Mobile Application

A cross-platform mobile application built with **Flutter**, developed as part of the Mobile Application Development (Flutter) internship task. The app replicates the authentication flow and dashboard experience of the Intern Edu website, converting it into a native Android/iOS mobile app.

---

## 📱 Overview

Intern Edu Mobile App provides a smooth, modern, and responsive user experience covering:
- User registration and secure login (powered by Firebase Authentication)
- A dashboard styled to match the Intern Edu website's look and feel
- Password recovery via email
- Profile management

---

## ✨ Features

- **Splash Screen** — animated app intro with auto-login check
- **Login / Sign Up Choice Screen** — lets users choose to log in or create a new account
- **Login Screen** — email & password login with validation and password visibility toggle
- **Registration Screen** — Full Name, Email, Mobile Number, Password, Confirm Password with full form validation
- **Forgot Password Screen** — sends a real password-reset email via Firebase
- **Dashboard**
  - Welcome header with user's name and avatar
  - Sidebar navigation menu (Drawer) styled after the website
  - Overview stat cards (Active Courses, Internships, Live Training, Daily Status, Queries Raised)
  - Profile tab — view and edit profile details
  - Settings tab — notification/dark-mode preferences, change password, logout
- **Authentication** — powered by Firebase Authentication (Email/Password)
- **Local Profile Storage** — extra profile fields (name, mobile) stored locally via `SharedPreferences`, linked to the user's Firebase UID

## 🚀 Getting Started

### Prerequisites
- [Flutter SDK](https://flutter.dev/docs/get-started/install) installed
- Android Studio or VS Code with Flutter/Dart plugins
- A Firebase project ([console.firebase.google.com](https://console.firebase.google.com))

### 1. Clone the repository
```bash
git clone <your-repo-url>
cd entern_edu
```

### 2. Install dependencies
```bash
flutter pub get
```

### 3. Set up Firebase
1. Create a project in the [Firebase Console](https://console.firebase.google.com)
2. Enable **Authentication → Sign-in method → Email/Password**
3. Install the FlutterFire CLI and connect your project:
   ```bash
   dart pub global activate flutterfire_cli
   flutterfire configure
   ```
   This generates `lib/firebase_options.dart` automatically.

### 4. Run the app
```bash
flutter run
```

### 5. Build a release APK (optional)
```bash
flutter build apk --release
```
The APK will be generated at `build/app/outputs/flutter-apk/app-release.apk`.

## 🔑 Application Flow

```
Splash Screen
    │
    ▼
Login / Sign Up (choice)
    │
    ├──► Registration ──► Login ──► Dashboard
    │
    └──► Login ──► Dashboard
```

- New users register, then are redirected to the Login screen
- Returning users log in directly
- Already-logged-in users skip straight to the Dashboard on next app launch
- Forgot Password sends a reset link via Firebase to the user's registered email

---

## 🔐 Authentication Notes

- Authentication is handled entirely by **Firebase Authentication** (Email/Password provider)
- Extra profile fields (Full Name, Mobile Number) are stored **locally** on the device via `SharedPreferences`, keyed to the user's Firebase UID — this avoids requiring a paid Firestore/Blaze billing plan
- Passwords are never stored or handled directly by the app — Firebase manages all credential security
